## 2. ステータス

| 状態 | 説明 |
|---|---|
| 起票 | 新規登録、未対応 |
| 対応中 | 修正担当者がアサインされ作業中 |

## 3. 重要度

| レベル | 説明 |
|---|---|
| Critical | 認証障害・重大セキュリティ問題・データ破損 |
| High | 主機能（検索・投稿・閲覧）が利用できない |
| Medium | 一部条件で不具合が発生する |
| Low | UI 表示崩れ・誤字など軽微 |

## 6. 運用ルール

| ID | ルール |
|---|---|
| OPS-01 | バグ発見時は当日中に起票する |
| OPS-02 | 修正時は対応コミットIDを記録する |
| OPS-03 | 修正後は必ず再テストを実施する |
| OPS-04 | 同一事象の重複起票は禁止する |
| OPS-05 | 保留時は理由を概要欄へ記載する |
| OPS-06 | 関連機能ID・関連テストIDを必ず記録する |
