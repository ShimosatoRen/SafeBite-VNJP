# 3. DB設計

## 3.1 ER図

```mermaid
erDiagram

    AspNetUsers ||--o{ DangerSpots : posts
    AspNetUsers ||--o{ Comments : writes

    DangerSpots ||--o{ Comments : has

    DangerSpots ||--o{ DangerSpotTags : has
    DangerTags ||--o{ DangerSpotTags : assigned

    AspNetUsers {
        bigint Id
        varchar UserName
        varchar Email
        varchar PasswordHash
    }

    DangerSpots {
        bigint Id
        bigint UserId
        varchar Title
        text Description
        int DangerLevel
        decimal Latitude
        decimal Longitude
        varchar Address
        varchar ImagePath
        timestamp CreatedAt
    }

    Comments {
        bigint Id
        bigint DangerSpotId
        bigint UserId
        text Content
        timestamp CreatedAt
    }

    DangerTags {
        bigint Id
        varchar Name
    }

    DangerSpotTags {
        bigint DangerSpotId
        bigint DangerTagId
    }
```

# 4. テーブル定義

## 4.1 AspNetUsers

| カラム名 | 型 | NULL | 説明 |

| Id | bigint | No | ユーザーID |

# 7. バリデーション設計

## 7.1 ユーザー登録

| 項目 | 条件 |
|---|---|
| UserName | 必須、50文字以内 |
| Email | 必須、メール形式 |
| Password | 必須、8文字以上 |

---

## 7.2 危険地点投稿

| 項目 | 条件 |
|---|---|
| Title | 必須、100文字以内 |
| Description | 2000文字以内 |
| DangerLevel | 1〜5 |
| ImagePath | jpg / png のみ |

---

## 7.3 コメント投稿

| 項目 | 条件 |
|---|---|
| Content | 必須 |
| Content | 500文字以内 |

# 12. 危険度判定ロジック

| 条件 | 加点 |
|---|---|
| 信号なし | +2 |
| 歩道なし | +2 |
| バイク密集 | +3 |
| 夜暗い | +2 |
| 横断困難 | +3 |

## 判定基準

| 合計点 | 危険度 |
|---|---|
| 1〜2 | 1 |
| 3〜4 | 2 |
| 5〜6 | 3 |
| 7〜8 | 4 |
| 9以上 | 5 |
